# GitHub Release Management

Available operations:

1. **List releases**
   - Repository (owner/repo)
   - Include drafts and pre-releases

2. **Get latest release**
   - Repository
   - View the most recent published release

3. **Get release by tag**
   - Repository
   - Tag name

4. **Get release by ID**
   - Repository
   - Release ID

5. **Create release**
   - Repository
   - Tag name (version)
   - Target branch/commit
   - Release name
   - Body (release notes)
   - Draft (true/false)
   - Prerelease (true/false)
   - Assets to upload (optional)

6. **Update release**
   - Repository
   - Release ID
   - Update name, body, draft status, prerelease

7. **Delete release**
   - Repository
   - Release ID

8. **List release assets**
   - Repository
   - Release ID

9. **Upload release asset**
   - Repository
   - Release ID
   - File path
   - Asset name
   - Content type

10. **Get release asset**
    - Repository
    - Asset ID

11. **Update release asset**
    - Repository
    - Asset ID
    - Name or label

12. **Delete release asset**
    - Repository
    - Asset ID

13. **Download release asset**
    - Repository
    - Asset ID
    - Save to local path

14. **Generate release notes**
    - Repository
    - Tag name
    - Previous tag (optional)
    - Target branch
    - Auto-generate from commits

15. **Compare releases**
    - Repository
    - Base release tag
    - Head release tag
    - View changes between versions
