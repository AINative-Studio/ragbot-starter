# GitHub Issue Management

Available operations:

1. **List issues**
   - Repository (owner/repo)
   - Filters:
     * State (open, closed, all)
     * Assignee
     * Creator
     * Labels
     * Milestone
     * Sort (created, updated, comments)

2. **Get issue details**
   - Repository and issue number
   - View full issue information

3. **Create issue**
   - Repository
   - Title
   - Body (description)
   - Assignees (optional)
   - Labels (optional)
   - Milestone (optional)

4. **Update issue**
   - Issue number
   - Title
   - Body
   - State (open/closed)
   - Assignees
   - Labels
   - Milestone

5. **Close issue**
   - Issue number
   - Closing comment (optional)

6. **Reopen issue**
   - Issue number

7. **Add comment to issue**
   - Issue number
   - Comment text

8. **List issue comments**
   - Issue number
   - Sort by created/updated

9. **Edit issue comment**
   - Comment ID
   - New comment text

10. **Delete issue comment**
    - Comment ID

11. **Add labels to issue**
    - Issue number
    - Label names (array)

12. **Remove labels from issue**
    - Issue number
    - Label names

13. **Assign users to issue**
    - Issue number
    - Usernames (array)

14. **Remove assignees**
    - Issue number
    - Usernames

15. **Lock issue**
    - Issue number
    - Lock reason

16. **Unlock issue**
    - Issue number
